### 버그 수정
- 상품 리스트 복사 시 이미지 보호 기능 개선. (#15336)
  - 상품 리스트를 복사할 때 원본 상품의 이미지가 안전하게 유지돼요.
  - 복사 과정에서 발생할 수 있는 이미지 손실 문제가 해결됐어요.
- 수기주문에서 회원 장바구니 상품 2개 이상 추가 안정성 개선 (#15391)
  - 상품 결제수단이 통합설정이고 회원등급의 결제수단이 일부 제한된 경우에도, 수기주문에서 장바구니 상품을 2개 이상 선택해 추가할 수 있도록 개선되었어요.
- 환불 처리 시 무한루프(타임아웃) 발생 개선 (#15650)
  - 예치금/마일리지가 포함된 주문 환불 시, 환불 금액에 콤마가 포함된 값이 입력되면 안분 계산이 무한루프에 빠져 타임아웃이 발생하던 문제를 개선했습니다.
- 비회원 주문 취소복원 불가 현상 개선 (#15715)
  - 취소복원 사전검증에서 비회원 주문에도 예치금/마일리지 보유 검증이 적용되어
  - 복원이 차단되던 문제를 수정했습니다.
- 네이버 아이디 로그인 차단 현상 수정 (네이버 CallbackURL 검증 강화 대응) (#15726)
  - 네이버의 OAuth CallbackURL 검증 강화(strict match)로 발생한 네이버 아이디 로그인 불가 현상을 수정합니다.
  - redirect_uri를 등록값과 동일한 고정 URL로 전송하고, 동적값(referer/returnUrl)은 state 파라미터로 이관했습니다.
- 관리자 주문 상세보기 N+1 쿼리 제거 (취소 포함 주문의 배송비 합산 로직 개선) (#15798)
  - 취소(c1~c4) 이력이 있는 주문의 관리자 주문 상세보기 진입 시 배송행마다 반복되던 `es_orderHandle` / `es_orderDeliveryOriginal` 단건 조회를 `IN` 일괄 조회로 변경했습니다.
  - 주간 슬로우쿼리 P0(주 약 1,082회)로 지적된 N+1을 제거해, 취소행 N개당 `2N`회 발생하던 쿼리를 **2회 고정**으로 개선했습니다.
- 주문 사은품 hidden 필드(giveCnt/selectCnt/giftNo) 변조 검증 강화 (#15799)
  - 주문 시 사은품 지급수량·선택수량 hidden 값을 개발자도구로 변조해도 그대로 주문되던 문제를, 재조회한 유효 사은품 정보 기준으로 검증·정규화하도록 개선했습니다.
  - 지급수량은 지급조건으로 강제 정규화하고, 선택수량이 조건과 다르면 차단하며, 조건에 없는 사은품(giftNo) 변조는 제거합니다. (일반주문·정기결제 프론트/모바일 공통 적용)
- 토스페이 결제완료 중복 처리로 인한 알림톡 중복 발송 및 주문상태 정합성 불일치 개선 (#15857)
  - 토스페이 결제 시 결제창 복귀와 PG 콜백이 같은 주문의 결제완료 처리를 동시에 실행해 알림톡이 중복 발송되고 주문 상태 정합성이 불일치하던 현상을 개선합니다.
  - 결제완료 처리가 주문당 한 번만 실행되도록 변경됩니다.
- 상품 판매 통계 로그 정확도 개선 (#15926)
  - 상품 판매 카운팅을 확인하는 로그 메시지가 더 정확하게 표시되도록 개선됐어요.
